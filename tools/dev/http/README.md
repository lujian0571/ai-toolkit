# HTTP 请求模拟器

## 工具说明

HTTP 请求模拟器是一个用于发送和测试 HTTP 请求的在线工具。该工具支持多种 HTTP 方法，可以设置请求头和请求体，并查看完整的响应信息。完全在浏览器中运行，不会将用户数据发送到任何服务器，确保用户隐私安全。

## 核心功能

1. **HTTP 请求发送**：
   - 支持常见的 HTTP 方法（GET、POST、PUT、DELETE 等）
   - 可自定义请求头
   - 支持请求体内容

2. **响应信息展示**：
   - 显示状态码和状态文本
   - 显示请求耗时和响应大小
   - 展示响应体和响应头

3. **请求配置**：
   - 灵活的请求方法选择
   - 多行请求头设置
   - 可编辑的请求体内容

## 使用说明

### 基本使用

1. 选择 HTTP 方法
2. 输入请求 URL
3. 根据需要添加请求头（每行一个，格式为 Key: Value）
4. 对于 POST/PUT 等方法，可以添加请求体
5. 点击"发送请求"按钮
6. 查看响应信息，包括状态码、响应头和响应体

### 支持的 HTTP 方法

- **GET**：请求指定的资源
- **POST**：向指定资源提交数据
- **PUT**：更新指定资源
- **DELETE**：删除指定资源
- **PATCH**：对资源进行部分修改
- **HEAD**：只获取响应头
- **OPTIONS**：获取服务器支持的通信选项

## 实现要点

1. 使用 Fetch API 发送 HTTP 请求
2. 采用 Bootstrap 5 构建响应式界面
3. 完全在客户端运行，保护用户数据隐私
4. 符合工具集统一 UI 风格和导航结构

## 技术细节

### 请求处理

1. 使用原生 Fetch API 发送请求
2. 支持自定义请求头和请求体
3. 自动处理 JSON 格式的请求体
4. 显示完整的响应信息

### 响应展示

1. 状态码和状态文本
2. 请求耗时和响应大小
3. 响应体内容（支持文本格式）
4. 响应头信息

## 注意事项

1. 工具完全在浏览器中运行，受同源策略限制
2. 某些请求可能因 CORS 限制而失败
3. 请确保请求的 URL 是安全可靠的
4. 敏感信息不会被存储或上传

## 可扩展方向

1. 支持 Cookie 管理
2. 添加请求历史记录功能
3. 支持文件上传
4. 增加身份验证选项（Basic Auth、Bearer Token 等）
5. 提供响应格式化显示（JSON、XML 等）
