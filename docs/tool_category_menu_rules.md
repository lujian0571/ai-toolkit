# 工具分类与菜单规范

## 规则说明

为了保持工具平台的一致性和用户体验，所有工具应按照指定的分类归类，并在导航菜单中正确显示。

## 应用范围

此规则适用于所有新增工具的分类和菜单项设置。

## 实施规范

1. 工具应按照功能类型归类到相应的分类中
2. 每个工具在导航菜单中应有明确的分类归属
3. 工具页面应使用标准的面包屑导航

## 标准工具分类

1. 加密解密
   - 包含各类加解密工具，如Base64、MD5、AES、RSA等

2. 编码解码
   - 包含各类编码解码工具，如URL编码、Hex编码、Unicode编码等

3. 文本处理
   - 包含文本处理工具，如大小写转换、排序去重、文本替换等

4. 数据转换
   - 包含数据格式转换工具，如进制转换等

5. 二维码工具
   - 包含二维码相关工具，如二维码生成、二维码识别等

6. 生成工具
   - 包含各类生成器工具，如密码生成器等

7. 开发工具
   - 包含开发者常用工具，如正则表达式测试器、Cron表达式解析器等

## 菜单项要求

1. 每个工具应在对应的分类下有唯一的菜单项
2. 菜单项名称应简洁明了，准确反映工具功能
3. 菜单项应链接到工具的主页面

## 面包屑导航规范

1. 工具页面应使用面包屑导航
2. 面包屑应包含两级：分类名称 / 工具名称
3. 面包屑不应包含首页链接
4. 当前页面应为激活状态（active）

## 示例

### 菜单结构示例
```
- 加密解密
  - Base64
  - MD5
  - AES
- 二维码工具
  - 二维码生成
  - 二维码识别
- 生成工具
  - 密码生成
```

### 面包屑示例
```
<nav aria-label="breadcrumb">
    <ol class="breadcrumb">
        <li class="breadcrumb-item">生成工具</li>
        <li class="breadcrumb-item active" aria-current="page">密码生成</li>
    </ol>
</nav>
```

## 注意事项

1. 新增工具必须遵循现有的分类体系，如有特殊需求需要新建分类，应提前讨论确定
2. 工具名称应避免过长，建议控制在10个字符以内
3. 菜单项应按功能相关性分组，同类型工具应放在一起
