# 分组设置区域布局规范

## 规则说明

为了提升工具页面的用户体验和界面一致性，当工具包含多个明显可区分的设置分组时，应采用统一的布局规范。该规范旨在帮助用户清晰地区分不同的设置区域，并建立一致的操作流程认知。

## 应用范围

此规则适用于所有包含多个设置分组的工具页面，特别是那些设置项较多、功能较复杂的工具。

## 实施规范

1. **卡片结构**：
   - 使用单一卡片包含所有相关设置分组
   - 卡片标题应概括性地描述整体功能，如"图片处理设置"、"数据转换参数"等

2. **区域划分**：
   - 相关设置应按功能逻辑分组
   - 使用视觉分割线将不同的设置分组分开
   - 每个分组应有清晰的标识

3. **分割线规范**：
   - 使用带图标的中心对齐分割线进行视觉区分
   - 分割线应具有良好的视觉层次感，清楚地表明不同分组的界限
   - 分割线中间应包含描述性文本和适当的图标

## 标准实现代码

```html
<!-- 卡片结构 -->
<div class="card mb-4">
    <div class="card-header">
        <h5 class="mb-0">功能描述性标题</h5>
    </div>
    <div class="card-body">
        <!-- 第一个设置分组 -->
        <div class="mb-4">
            <h6>分组1标题</h6>
            <!-- 分组1的设置项 -->
        </div>
        
        <!-- 视觉分割线 -->
        <div class="d-flex align-items-center my-4">
          <div class="flex-grow-1 border-top"></div>
          <span class="mx-3 text-muted">
            <i class="bi bi-gear-fill me-1"></i>
            分组2标题
          </span>
          <div class="flex-grow-1 border-top"></div>
        </div>
        
        <!-- 第二个设置分组 -->
        <div class="mb-4">
            <h6>分组2标题</h6>
            <!-- 分组2的设置项 -->
        </div>
        
        <!-- 操作按钮 -->
        <button class="btn btn-primary">
            <i class="bi bi-play-fill me-1"></i>
            执行操作
        </button>
    </div>
</div>
```

## 设计原则

1. **用户流程引导**：
   - 用户按照从上到下的顺序依次查看各个设置分组
   - 分割线作为一个视觉停顿点，帮助用户意识到接下来是新的设置分组
   - 每个分组内的设置项应该逻辑相关
   - 最后是执行操作按钮

2. **视觉层次**：
   - 不同设置分组应有明显的视觉区分
   - 分割线应足够突出，但不过于显眼以至于分散注意力
   - 图标应与文本内容语义一致

3. **响应式设计**：
   - 布局在不同屏幕尺寸下应保持良好的可读性和可用性
   - 分割线在小屏幕上也应正常显示

## 使用场景示例

1. 图片处理工具（如尺寸调整、格式转换、压缩、水印等）
2. 文档处理工具（如格式转换、加密解密等）
3. 数据处理工具（如格式转换、清洗、分析等）
4. 任何包含多个逻辑分组设置项的工具

## 注意事项

1. 分割线文本应根据具体分组功能进行适当调整
2. 图标可根据具体语义进行替换，但应保持在整个项目中的一致性
3. 设置分组应按照用户的自然操作习惯进行排列
4. 对于设置项特别多的情况，可考虑使用折叠面板或选项卡进行进一步组织
5. 同一分组内的设置项应保持逻辑相关性
